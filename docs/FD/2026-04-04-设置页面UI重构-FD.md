## 1. 页面结构总览

### 1.1 页面层次

```
#page-settings
├── .settings-tab-nav                    // Tab 导航栏
│   ├── .settings-tab[data-tab="basic"]          // 基础
│   ├── .settings-tab[data-tab="temp-mail"]      // 临时邮箱
```

| 现状 | 重构后 |

| `max-width: 720px` | 宽度限制移除 |

## 2. Tab 导航栏组件规格

### 2.1 DOM 结构

```html
<div class="settings-tab-nav" id="settingsTabNav">
    <button class="settings-tab active" data-tab="basic"
            onclick="switchSettingsTab('basic')">基础</button>
    <button class="settings-tab" data-tab="temp-mail"
            onclick="switchSettingsTab('temp-mail')">临时邮箱</button>
    <button class="settings-tab" data-tab="api-security"
            onclick="switchSettingsTab('api-security')">API 安全</button>
    <button class="settings-tab" data-tab="automation"
            onclick="switchSettingsTab('automation')">自动化</button>
</div>
```

### 2.2 视觉规格

| 状态 | 样式 |
|------|------|
| 默认（未选中） | 文字颜色 `var(--text-muted)`，字重 500 |
| Hover | 文字颜色 `var(--text)` |
| 选中（active） | 文字颜色 `var(--clr-primary)`，字重 600，底部 2px 实线 `var(--clr-primary)` |
| 焦点 | 保持浏览器默认 outline（可访问性） |

### 2.3 布局规格

- 水平排列，`gap: 0`（紧靠）
- 整行底部有 2px 分隔线 `var(--border-light)`
- 窄屏（< 480px）支持横向滚动（`overflow-x: auto`），不折行
- 无 Emoji，纯文字
- Tab 按钮内边距：`0.75rem 1.5rem`

### 2.4 保存状态指示（切换时）

- 触发自动保存时，对应 Tab 按钮文字后追加小圆点（`·`）变为 `var(--clr-warn)` 色
- 保存完成后圆点消失
- 保存失败后圆点变为 `var(--clr-danger)` 色并保留（直至用户手动触发保存成功）

> 实现方式：JS 动态给 Tab button 添加/移除 `.saving` / `.save-error` class

## 5. 只读字段规格

适用于：`settingsCfWorkerDomains`、`settingsCfWorkerDefaultDomain`

### 5.1 视觉

| 属性 | 值 |
|------|------|
| 背景色 | `var(--bg-secondary)` |
| 边框色 | `var(--border-light)` |
| 文字颜色 | `var(--text-muted)` |
| 光标 | `cursor: default` |
| 右侧装饰 | 🔒 图标（通过 wrapper `position: relative` + `::after` 实现） |
| HTML 属性 | `readonly` + JS 设置 `.readOnly = true` |

### 5.2 交互

- 不可点击输入（`pointer-events` 不阻断但 `readOnly` 生效）
- Hover 时显示 tooltip：「由 CF Worker 同步，不可手动修改」
- 「保存设置」按钮触发时，忽略这两个字段，不写入 PUT 请求

### 5.3 状态变化时机

| 时机 | 字段状态变化 |
|------|------|
| 页面初始化 → 加载 settings，`cf_worker_domains` 非空 | 只读，显示已同步的域名 JSON |
| 页面初始化 → 加载 settings，`cf_worker_domains` 为空数组 | **仍然只读**，字段为空，显示 placeholder：「尚未同步，请点击上方按钮同步」 |
| 同步按钮点击成功 | 更新字段内容 → 维持只读状态（字段从始至终只读） |

**设计原则**：CF Worker 域名字段**永久只读**（由 PRD 确认）。用户无论何时都不能手动编辑该字段，只能通过同步按钮触发写入。字段为空时通过 placeholder 文字引导用户操作。

## 6. 交互流程

### 6.1 Tab 切换流程

```
用户点击 Tab [B]
    ↓
检查当前 Tab [A] 是否为「基础」Tab
    ├─ 是「基础」Tab
    │       ↓
    │   检查密码框是否有内容
    │       ├─ 有内容 → 清空密码框 + 显示提示 Banner（密码未保存提示，5 秒自动消失）
    │       └─ 无内容 → 无需处理
    │       ↓
    │   立即切换到 Tab [B]（不触发自动保存）
    │
    └─ 不是「基础」Tab
            ↓
        【立即】切换到 Tab [B]（视觉立即响应，不等待保存）
            ↓
        后台异步触发自动保存（autoSaveSettings(prevTab)）
            ├─ 保存中 → Tab [A] 按钮显示 · 圆点（.saving class，黄色）
            ├─ 保存成功 → 圆点消失
            └─ 保存失败 → 圆点变红并保留（.save-error class）
                          + 页面顶部显示持久错误 Banner（不自动消失）
```

> **时序说明（v0.3 新增）**：Tab 切换与自动保存是**并行关系**，不是串行关系。用户点击 Tab 后内容区**立即切换**，保存在后台异步进行，不阻塞 UI。保存圆点为后台进度反馈，不影响用户继续操作。

### 6.2 自动保存边界规则

| Tab | 参与自动保存的字段 | 排除的字段 |
|------|------|------|
| 基础 | **无**（基础 Tab 不触发自动保存） | `settingsPassword`（密码） |
| 临时邮箱 | `temp_mail_provider` + 当前可见面板的可编辑字段 | 隐藏面板的字段；CF Worker 只读域名字段 |
| API 安全 | 本 Tab 所有字段 | — |
| 自动化 | 本 Tab 所有字段 | — |

### 6.3 Provider 切换流程

```
用户选中 Provider 单选按钮
    ↓
触发 onTempMailProviderChange(value)
    ├─ value == 'legacy_bridge'
    │   → gptmailConfigPanel: display: block
    │   → cfWorkerConfigPanel: display: none
    │
    └─ value == 'cloudflare_temp_mail'
        → gptmailConfigPanel: display: none
        → cfWorkerConfigPanel: display: block
        → 若 cf_worker_domains 已加载且非空，只读字段状态已就绪，无需额外操作
```

### 6.4 CF Worker 同步流程

```
用户点击「从 CF Worker 同步域名」按钮
    ↓
按钮进入 loading 状态（disabled + 显示「同步中...」）
    ↓
调用 /api/settings/cf-worker-sync-domains（POST）
    ├─ 成功
    │   → 更新 settingsCfWorkerDomains 字段内容（只读 JSON 展示）
    │   → 更新 settingsCfWorkerDefaultDomain 字段内容（只读）
    │   → 更新 #cfWorkerSyncTime 显示「上次同步：{时间}」
    │   → 按钮恢复可用
    │
    └─ 失败
        → 显示错误 Toast（不清空字段内容）
        → 按钮恢复可用
        → 字段状态不变
```

### 6.5 页面初始化（loadSettings）流程

```
进入设置页面 → 调用 loadSettings()
    ↓
GET /api/settings
    ↓
填充所有字段（按现有逻辑）
    ↓
处理新增字段：
    ├─ temp_mail_provider → 设置 radio button 选中状态
    │   → 触发 onTempMailProviderChange(provider) 显隐面板
    ├─ cf_worker_domains → 填充 #settingsCfWorkerDomains
    │   → 若非空 → 设为只读
    ├─ cf_worker_default_domain → 填充 #settingsCfWorkerDefaultDomain
    │   → 若非空 → 设为只读
    └─ cf_worker_prefix_rules → 填充 #settingsCfWorkerPrefixRules（可编辑）
```

## 7. 错误提示 Banner 规格

### 7.1 密码未保存提示（非持久）

- 触发条件：「基础」Tab 密码框有内容时，用户切换 Tab
- 位置：页面顶部固定，全宽
- 样式：黄色警告色（`var(--clr-warn)` 背景）
- 内容：「密码修改未保存，如需修改请在「基础」Tab 重新输入后点击保存」
- 行为：显示后 5 秒自动消失（或用户点击 × 关闭）

### 7.2 自动保存失败提示（持久）

- 触发条件：Tab 切换触发自动保存，保存请求失败
- 位置：页面顶部固定，全宽
- 样式：红色错误色（`var(--clr-danger)` 背景）
- 内容：「保存失败，[临时邮箱] Tab 的修改尚未保存，请手动重试」
- 行为：**不自动消失**，直至用户手动关闭（× 按钮）或该 Tab 保存成功

### 7.3 复用现有 Toast 系统 vs 新增 Banner

**决策（已对齐）：扩展现有 `showToast` 函数，增加 `persistent` 参数。**

- 「密码未保存」：`showToast(message, 'warning')` — **5 秒**自动消失（与 §7.1 一致）
- 「自动保存失败」：`showToast(message, 'error', null, true)` — `persistent=true`，Toast 内追加 × 关闭按钮，不自动消失

扩展方案详见 TD §3.5（`showToast` 持久模式改造）。

## 8. 保存按钮策略

### 8.1 每个 Tab 独立保存按钮

每个 Tab 内均保留「保存设置」按钮，用户可随时手动触发保存，无需依赖 Tab 切换。

### 8.2 保存范围

手动保存时，保存范围与自动保存相同（见 §6.2），即只保存当前 Tab 可见的可编辑字段：

- 基础 Tab：仅保存密码（非空时）
- 临时邮箱 Tab：保存 `temp_mail_provider` + 当前可见 Provider 面板可编辑字段
- API 安全 Tab：保存本 Tab 所有字段
- 自动化 Tab：保存本 Tab 所有字段

### 8.3 现有 saveSettings() 函数改造方向

现有 `saveSettings()` 收集全部字段后一次性 PUT，需改造为：
1. 检测当前激活的 Tab
2. 仅收集该 Tab 对应的字段子集
3. 调用 PUT `/api/settings`

详细实现见 TD §3.3。
