## Phase 1：提取统一轮询引擎（不改现有功能）

**目标**：将 `mailbox_compact.js` 中的轮询引擎核心代码提取到独立文件 `poll-engine.js`，`mailbox_compact.js` 改为调用引擎 API。此阶段**不改变任何外部行为**，纯内部重构。

- [ ] **P1-1**：新建 `static/js/features/poll-engine.js`
  - 迁移以下函数/变量：
    - `compactPollMap` → `pollMap`
    - `compactPollCountdownTimer` → `pollCountdownTimer`
    - `COMPACT_POLL_TOAST_DURATION` → `POLL_TOAST_DURATION`
    - `COMPACT_POLL_INITIAL_DELAY_MS` → `POLL_INITIAL_DELAY_MS`
    - `compactT()` → `pollT()`（翻译函数）
    - `_handlePollError()` → 保留
    - `_notifyNewEmailAndStop()` → 保留
    - `stopCompactAutoPoll()` → `stopPoll(email, toastMsg, toastType)`
    - `stopAllCompactAutoPolls()` → `stopAllPolls()`
    - `startGlobalCountdown()` → 保留
    - `pollSingleEmail()` → 保留
    - `startCompactAutoPoll()` → `startPoll(email, opts)`
    - `applyCompactPollSettingsToRunningPolls()` → `applyPollSettingsToRunning(newSettings)`
    - `applyCompactPollSettings()` → `applyPollSettings(settings)`
    - `visibilitychange` 监听 → 保留
  - **不迁移**（保留在 `mailbox_compact.js`）：
    - `findCompactAccountRow()` — compact-specific DOM 查找
    - `updateCompactPollUI()` — compact-specific UI 更新
    - `updateSingleRowFromCache()` — compact-specific 行更新
    - `reapplyAllCompactPollUI()` — compact-specific UI 恢复
    - `renderCompactAccountList()` — compact 列表渲染
  - 引擎需支持 **UI 回调注入**：
    - `onPollStart(email, maxCount)` — 轮询开始时更新 UI
    - `onPollStop(email)` — 轮询停止时恢复 UI
    - `onPollTick(email, remainingCount)` — 每秒倒计时更新 UI
    - `onNewEmail(email, summary)` — 发现新邮件时更新缓存/UI

- [ ] **P1-2**：`mailbox_compact.js` 改为调用 `poll-engine.js` API
  - `email-copied` 事件监听改为调用 `startPoll(email, { mode: 'compact' })`
  - 注册 compact UI 回调：
    ```js
    registerPollCallbacks({
      onPollStart: (email, maxCount) => updateCompactPollUI(email, 'polling', maxCount),
      onPollStop: (email) => updateCompactPollUI(email, 'stopped', null),
      onPollTick: (email, remaining) => updateCompactPollUI(email, 'polling', remaining),
      onNewEmail: (email, summary) => updateSingleRowFromCache(email, summary),
    });
    ```

- [ ] **P1-3**：`templates/partials/scripts.html` 中引入 `poll-engine.js`（在 `mailbox_compact.js` 之前）

- [ ] **P1-4**：运行 TC-C（Jest 引擎测试）确认全绿
  - `tests/compact-poll/setup.js` 需要也 eval `poll-engine.js`

- [ ] **P1-5**：运行 TC-A + TC-B 确认不回归

- [ ] **P1-6**：提交 Phase 1

## Phase 2：标准模式接入统一引擎

**目标**：删除 `main.js` 中的旧轮询代码（~150 行），标准模式改为使用 `poll-engine.js`。

- [ ] **P2-1**：删除 `main.js` 中以下函数/变量：
  - `let pollingTimer`、`let isPolling`、`let pollingCount`、`let maxPollingCount`、`let pollingInterval`、`let autoPollingEnabled`
  - `let knownEmailIds`、`let pollingErrorCount`、`MAX_POLLING_ERRORS`
  - `function startPolling()`
  - `function stopPolling()`
  - `function pollForNewEmails()`
  - `function showPollingStatusIndicator()` / `hidePollingStatusIndicator()`
  - `function showNewEmailNotification()`
  - `function showAccountNewEmailDot()` / `hideAccountNewEmailDot()`
  - `function showNewEmailIndicator()` / `hideNewEmailIndicator()`
  - `function syncPollingForCurrentAccount()`
  - `function initPollingSettings()` 中的标准轮询初始化

- [ ] **P2-2**：`main.js` 注册标准模式 UI 回调
  ```js
  registerPollCallbacks({
    onPollStart: (email, maxCount) => showPollingDot(email),
    onPollStop: (email) => hidePollingDot(email),
    onPollTick: null,  // 标准模式不需要倒计时文字
    onNewEmail: (email, summary) => showNewEmailToast(email, summary),
  });
  ```

- [ ] **P2-3**：`emails.js` 中 `email-copied` 事件已由引擎统一监听，不需要额外改动

- [ ] **P2-4**：删除 `groups.js` 中 `renderAccountList` 末尾的 `showPollingStatusIndicator` 调用

## 验收标准

- [ ] 只有 1 个轮询设置面板（标准轮询面板，控制所有模式）
- [ ] 标准模式和简洁模式都通过"复制邮箱"触发轮询

- [ ] 发现新邮件后自动提取验证码并复制到剪贴板

- [ ] 页面切到后台时暂停轮询，切回时恢复
- [ ] 所有测试全绿（预计 40-49 个）
- [ ] 代码总量减少（目标：净减少 100+ 行）

## 注意事项

1. **Phase 1 是纯内部重构**，不应改变任何外部行为。如果 Phase 1 后测试不全绿，说明迁移有问题，应立即修复。
2. **Phase 2 有行为变更**（标准模式触发方式改变），需要人工验证。
3. **Phase 3 是设置层面的破坏性变更**，compact 字段被废弃。但由于 Phase 2 已经让引擎不再使用这些字段，影响可控。
4. 每个 Phase 完成后都应提交一个独立 commit，方便回滚。
