## 1. 文档目标

本 TDD 只关注"浏览器扩展邮箱池快捷操作面板"的测试设计，不重复展开业务背景与技术方案。

本次测试要证明七件事：

1. 主应用 CORS 改动后，来自 `chrome-extension://` 来源的请求能正常通过，现有来源不受影响
2. Storage 封装函数的读写语义正确（含 history 最多 100 条限制）
3. 申领流程：task_id 先写 storage 再发 API，申领失败后 currentTask 被清空
4. 验证码/链接获取超时后，状态回到 `claimed`（任务不被清空，可重试）
5. 完成/释放后，即使 API 失败，currentTask 也被清空，history 正确标注 apiError
6. Popup 重新打开时，正确恢复 currentTask（有任务则显示 claimed，无任务则 idle）
7. 设置面板：权限申请用户拒绝时，配置不被保存

## 3. 测试原则

1. **主应用侧变更必须自动化**：CORS 改动涉及现有 API 安全边界，必须有自动化测试锁定，防止后续回归。
2. **扩展侧优先手工冒烟**：v0.1.0 阶段扩展侧逻辑可用手工验证，后续按需引入 Jest。
3. **不测界面像素，测行为**：JS 单元测试关注函数返回值和 storage 写入内容，不关注 DOM 渲染细节。
4. **异常路径也要测**：申领失败、获取超时、API 调用失败（完成/释放时）这三条异常路径是核心验证点。

## 5. A 层：主应用 CORS 测试设计

建议新建：`tests/test_extension_cors.py`

### 5.1 测试结构

```python
class TestChromeExtensionCORS(unittest.TestCase):
    """测试主应用 CORS 支持 chrome-extension:// 来源"""

    def setUp(self):
        self.app = create_app(testing=True)
        self.client = self.app.test_client()
        self.api_key = "test-api-key"  # 与现有测试一致的测试 Key

    def _make_cors_request(self, method, path, origin, **kwargs):
        headers = kwargs.pop('headers', {})
        headers['Origin'] = origin
        headers['X-API-Key'] = self.api_key
        return getattr(self.client, method.lower())(path, headers=headers, **kwargs)

    def _make_preflight(self, path, origin, request_method='POST'):
        return self.client.options(path, headers={
            'Origin': origin,
            'Access-Control-Request-Method': request_method,
            'Access-Control-Request-Headers': 'Content-Type, X-API-Key',
        })
```

### 5.2 关键测试方法

```python
def test_claim_random_allows_chrome_extension_origin(self):
    """CR-01: POST claim-random 允许 chrome-extension:// 来源"""
    resp = self._make_cors_request(
        'POST', '/api/external/pool/claim-random',
        origin='chrome-extension://abcdef1234567890',
        json={'caller_id': 'browser-extension', 'task_id': 'test-uuid'}
    )
    # 无论 API 本身成功/失败（鉴权等），CORS headers 应当存在
    assert 'Access-Control-Allow-Origin' in resp.headers
    acao = resp.headers['Access-Control-Allow-Origin']
    assert acao == 'chrome-extension://abcdef1234567890' or acao == '*'

def test_preflight_allows_x_api_key_header(self):
    """CR-02: OPTIONS 预检包含 X-API-Key 在 Access-Control-Allow-Headers"""
    resp = self._make_preflight('/api/external/pool/claim-random',
                                 origin='chrome-extension://abcdef1234567890')
    assert resp.status_code in [200, 204]
    acah = resp.headers.get('Access-Control-Allow-Headers', '')
    assert 'X-API-Key' in acah or 'x-api-key' in acah.lower()

def test_verification_code_allows_chrome_extension_origin(self):
    """CR-03: GET verification-code 允许 chrome-extension:// 来源"""
    resp = self._make_cors_request(
        'GET', '/api/external/verification-code?email=test@example.com',
        origin='chrome-extension://abcdef1234567890'
    )
    assert 'Access-Control-Allow-Origin' in resp.headers

def test_cors_does_not_affect_existing_origin(self):
    """CR-07: 改动不影响已有来源的 CORS 行为"""
    # 现有调用方模拟（如同域或已有白名单来源）
    resp = self._make_cors_request(
        'POST', '/api/external/pool/claim-random',
        origin='https://legitimate-caller.example.com',
        json={'caller_id': 'test', 'task_id': 'test-uuid'}
    )
    # 不应因为新增 chrome-extension 支持而破坏已有来源响应

def test_chrome_extension_origin_not_allowed_on_internal_api(self):
    """CR-08: chrome-extension 来源不被内部 Session API 允许"""
    resp = self.client.get('/api/accounts/', headers={
        'Origin': 'chrome-extension://abcdef1234567890'
    })
    # 内部 API 不应返回 Access-Control-Allow-Origin: chrome-extension://...
    acao = resp.headers.get('Access-Control-Allow-Origin', '')
    assert 'chrome-extension://' not in acao
```

### 5.3 关键断言说明

- 断言 `Access-Control-Allow-Origin` 存在且包含 chrome-extension 来源
- 断言 OPTIONS 预检响应码为 200 或 204
- 断言 `X-API-Key` 在允许的 headers 中（大小写不敏感）
- 断言内部 API 不受影响
