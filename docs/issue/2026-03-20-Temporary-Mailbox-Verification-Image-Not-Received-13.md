- 主题：bug: 临时邮箱收到的验证码接收不到图片 #13

## 问题描述

当前在使用临时邮箱接收验证码邮件时，邮件本身可以到达，但验证码相关图片内容无法正常接收或显示。

从使用表现看，这不是“整封邮件收不到”，而是“邮件里的图片资源没有成功呈现”，导致用户无法完整查看验证码内容或依赖图片中的关键信息。

## 当前影响

- 用户虽然收到了验证码邮件，但无法看到邮件中的图片内容。
- 如果验证码、校验图或关键提示信息依赖图片展示，当前流程会直接受阻。
- 临时邮箱场景下的可用性下降，影响验证码读取成功率。

## 可能表现形式

- 邮件正文中的图片为空白。
- 图片链接失效或未被前端渲染。
- 邮件解析后只保留了文本，图片资源未保留。
- 图片为内嵌资源时，CID 或 base64 内容未被正确处理。

## 初步判断

高概率问题位于“邮件内容解析”或“前端展示”链路，而不是邮件接收链路本身。

需要重点检查：

- 临时邮箱来源的邮件 HTML 是否被完整保存
- 邮件正文中的 `img`、`cid:`、`data:`、远程图片链接是否被过滤或丢失
- 后端返回给前端的邮件内容是否只保留了纯文本
- 前端渲染邮件详情时是否对图片标签、图片地址或富文本做了额外清洗

## 建议修复方向

1. 对临时邮箱收到的原始邮件 HTML 做一次完整抓取和比对，确认图片资源在后端入库前是否已丢失。
2. 检查邮件解析逻辑，确认对内嵌图片、CID 引用、base64 图片和远程图片链接都有兼容处理。
3. 检查前端邮件详情页的渲染逻辑，确认没有把图片标签或相关属性过滤掉。
4. 若出于安全策略限制了外链图片，需要补充明确提示或提供“允许加载图片”的机制。

## 建议验收标准

1. 临时邮箱收到包含图片的验证码邮件后，邮件详情中可以正常看到图片。
2. 对包含内嵌图片、CID 图片和远程图片的邮件都能正确展示主要内容。
3. 图片无法加载时，界面可以给出明确提示，而不是静默缺失。
4. 修复后不影响普通文本邮件和已有邮件读取能力。
