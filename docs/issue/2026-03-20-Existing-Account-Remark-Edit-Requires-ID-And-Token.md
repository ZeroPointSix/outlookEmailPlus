# Issue 记录

## 基本信息

- 记录日期：2026-03-20
- 状态：待修复
- 优先级：P2
- 主题：已有账号编辑备注时仍要求补全账号 ID 和 token

## 问题描述

当前系统在为“已存在的账号”补充或修改备注时，仍要求用户补全账号 ID 和 token。

这个行为不符合“轻量资料编辑”的预期，因为备注本质上只是账号的附加说明字段，不应被当成一次完整的账号重提交流程处理。

## 当前影响

- 用户无法只针对备注字段做独立更新。
- 编辑备注时需要再次处理无关的账号凭据信息，增加操作成本。
- 如果用户手头没有当前账号的 ID 或 token，备注编辑可能直接失败。
- 前端和后端会把“备注更新”与“完整账号信息校验”耦合在一起，影响后续扩展更多轻量编辑能力。

## 预期行为

- 针对已有账号编辑备注时，应允许只提交备注相关字段。
- 备注更新不应要求用户重新填写账号 ID。
- 备注更新不应要求用户重新填写 token。
- 保存时仅更新备注字段，不覆盖账号、密码、token、状态等其他信息。

## 初步判断

高概率是当前备注编辑复用了“新增账号”或“完整编辑账号”的校验逻辑，导致备注字段更新也被要求补全整套账号信息。

需要重点检查：

- 前端备注编辑入口是否直接复用了完整账号表单
- 后端备注更新接口是否复用了新增/全量更新校验
- 请求 DTO / 表单校验层是否把 ID、token 设为了备注更新时的必填项

## 建议修复方向

1. 将“备注更新”从“完整账号编辑”中拆分出来，作为单独的轻量更新行为处理。
2. 为已有账号提供单字段或部分字段更新接口，避免复用全量校验。
3. 前端在备注编辑场景下只提交备注相关字段和定位已有账号所需的最小标识。
4. 后端对备注更新场景使用独立校验规则，不再要求 token 等无关字段。

## 建议验收标准

1. 选择一个已有账号后，可以直接新增备注，无需补全 token。
2. 选择一个已有账号后，可以直接修改备注，无需补全 token。
3. 清空备注后可以成功保存。
4. 保存备注后，仅备注字段发生变化，其它账号信息保持不变。
