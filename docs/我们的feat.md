# 我们的 feat

## 新的功能需求

### 1. 支持为已有账号添加额外备注功能

#### 背景

当前账号信息以登录信息和基础状态为主，缺少一个专门用于补充业务说明的轻量备注能力。实际使用中，用户可能需要给账号补充来源、用途、使用说明、风险标记或人工备注，因此需要一个独立的“额外备注”字段。

#### 需求描述

- 为已有账号新增一个“额外备注”编辑能力。
- 备注字段用于记录业务侧补充说明，不影响账号核心凭据字段。
- 支持在账号列表或详情中查看当前备注内容。
- 支持用户对备注进行新增、修改和清空。
- 保存备注时，只更新备注相关字段，不覆盖账号的其他已有信息。

#### 期望行为

- 已有账号进入备注编辑后，可以直接新增、修改或清空备注。
- 备注可以作为账号的补充信息长期保存，并在后续管理过程中持续可见。
- 备注功能应与账号新增、导入、状态管理等主流程解耦，作为独立的信息维护能力存在。

#### 验收标准

- 选择一个已有账号后，可以新增额外备注。
- 已存在备注时，可以再次修改并保存。
- 已存在备注时，可以清空备注并保存。
- 提交备注后，只有备注字段发生变化，账号、密码、token、状态等其他字段保持不变。
- 备注支持新增、修改和清空三种操作。

当前“编辑备注时仍需补全账号 ID 和 token”的问题，已单独迁移到 `docs/issue` 目录作为 issue 记录，不再作为本功能需求本身描述。
